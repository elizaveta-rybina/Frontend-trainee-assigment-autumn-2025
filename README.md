## Быстрый старт

1. **Клонируем репозиторий**

```sh
git clone https://github.com/elizaveta-rybina/Frontend-trainee-assigment-autumn-2025.git
cd Frontend-trainee-assigment-autumn-2025
```

2. **Сборка Docker-образов**

### Backend

```sh
docker build -f ./backend/Dockerfile ./backend
```

### Frontend

```sh
docker build -f ./frontend/Dockerfile ./frontend
```

3. **Запуск проекта в режиме разработки**

```sh
docker compose up --build -d
```

Эта команда поднимет все сервисы (frontend и backend) в фоновом режиме.

4. **Остановка контейнеров**

```sh
docker compose down
```

## Запуск в разработке

1. **Установить зависимости**

```sh
cd frontend
npm install
```

2. **Запустить фронтенд в режиме разработки**

```sh
npm run dev
```

По умолчанию Vite поднимет сервер на `http://localhost:5173`, и изменения в коде будут автоматически отражаться в браузере благодаря HMR (Hot Module Replacement).

3. **Backend**

Дальше необходимо запустить бэкенд:

```sh
docker build -f ./backend/Dockerfile ./backend
docker compose up -d
```

4. **Переменные окружения**

```sh
VITE_API_URL=http://localhost:3000/api/v1
```

VITE_API_URL — URL вашего backend API.

## Структура проекта (Frontend)

- **`src/`** — исходный код приложения:

  - **`app/`** — конфигурация приложения и маршрутизация:

    - **`api/`** — файлы для работы с API: типы, константы, хуки и функции для запросов.
    - **`providers/`** — провайдеры (для React Query).
    - **`routes/`** — маршруты приложения и их конфигурация.

  - **`components/`** — повторно используемые UI-компоненты:

    - **`DetailViewPage/`** — компоненты страницы просмотра объявления (навигация, модерация, виджеты и детали объявления).
    - **`MainPage/`** — компоненты главной страницы (фильтры, сортировка, поиск, список объявлений, пагинация).
    - **`StatsPage/`** — компоненты страницы аналитики (графики, метрики, выбор периода).

  - **`pages/`** — страницы приложения: MainPage, DetailViewPage, AnalyticsPage, NotFoundPage.
  - **`shared/`** — общие утилиты и UI-компоненты:

  - **`App.tsx`** — главный компонент приложения.
  - **`main.tsx`** — точка входа в приложение, подключение провайдеров и рендер в DOM.
  - **`index.css`** — глобальные стили.
  - **`setupTests.ts`** — настройка тестовой среды.
  - **`vite-env.d.ts` и `custom.d.ts`** — объявления типов TypeScript.

- **`package.json` / `package-lock.json`** — зависимости проекта.

- **`tsconfig*.json`** — конфигурации TypeScript.

- **`vite.config.ts`** — конфигурация Vite.

## Обоснование выбранных библиотек

- **React** — основной фреймворк для построения UI.
- **react-dom** — рендеринг React-компонентов в DOM.
- **react-router-dom** — маршрутизация между страницами приложения.
- **@tanstack/react-query** — удобная работа с серверными данными, кэширование и автоматическое обновление данных.
- **axios** — библиотека для HTTP-запросов к backend API.
- **chart.js** и **react-chartjs-2** — построение графиков и визуализация статистики.
- **lucide-react** — готовые SVG-иконки для интерфейса.
- **clsx** — удобная работа с динамическими классами CSS.
- **sass** — использование SCSS для более удобного и структурированного написания стилей.
- **jest**, **@testing-library/react**, **@testing-library/jest-dom**, **@testing-library/user-event** — тестирование компонентов и логики приложения.
- **typescript** и связанные типы (`@types/react`, `@types/react-dom`, `@types/node`) — строго типизированный код для предотвращения ошибок.
- **vite** и **@vitejs/plugin-react** — быстрый сборщик и разработка проекта с поддержкой HMR.
- **eslint**, **eslint-plugin-react-hooks**, **eslint-plugin-react-refresh**, **typescript-eslint** — проверка кода и соблюдение единого стиля.
